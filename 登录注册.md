# 登录设计知识点
---
  1. 核心新增文件（最重要）

  src/proxy.ts

  - 替代了旧的 middleware.ts（Next.js 16 改名了）
  - 功能：路由保护，自动检查用户是否登录
    - 未登录访问 /todos → 跳转到 /login
    - 已登录访问 /login → 跳转到 /todos

  src/app/auth/actions.ts

  - Server Actions 处理登录注册表单
  - loginAction：调用 supabase.auth.signInWithPassword()
  - registerAction：调用 supabase.auth.signUp()
  - logoutAction：登出并跳转

  src/app/login/

  - page.tsx：登录页面入口
  - _components/auth-form.tsx：使用 Tabs 切换登录/注册
  - 用 useActionState 处理表单提交状态

  src/lib/supabase.ts + src/lib/supabase.server.ts

  - 用 @supabase/ssr 替代了旧的 supabase-js 客户端
  - supabase.ts：浏览器端，管理会话 cookie
  - supabase.server.ts：服务端（Server Actions/Components）

  src/components/providers/auth-provider.tsx

  - 监听 supabase.auth.onAuthStateChange()
  - 更新 zustand store 里的用户信息
  - 让 UI 能响应登录/登出状态

  src/stores/auth-store.ts

  - 全局状态：user
  - 显示当前用户邮箱、登出按钮等

  ---
  2. 修改的文件

  src/app/todos/_lib/todo-service.ts

  - 所有函数新增 userId 参数
  - 查询/插入/更新/删除都加了 user_id 过滤
  - 用 @supabase/ssr 的服务端客户端

  src/app/todos/actions.ts

  - 顶部先通过 createServerClient().auth.getUser() 获取当前用户
  - 每个 action 都依赖用户 ID
  - 未登录时不执行操作

  src/app/todos/page.tsx

  - Server Component，获取当前用户
  - 传 userId 给 getTodos(userId)

  src/app/todos/_components/todo-header.tsx

  - 显示当前用户邮箱
  - 添加"退出"按钮（调用 logoutAction）

  src/app/layout.tsx

  - 包裹 AuthProvider
  - 添加 sonner 的 <Toaster />

  supabase/schema.sql

  - todos 表新增 user_id 列，外键关联 auth.users(id)
  - 删除匿名开放策略
  - 新增用户隔离策略：auth.uid() = user_id

  ---
  3. 其他新增

  scripts/seed-admin.mjs

  - 创建默认 admin 用户
  - 把旧的匿名 todos 数据关联到 admin

  package.json

  - 新增依赖：@supabase/ssr

  ---
  总结改动思路

  1. 用 @supabase/ssr 统一浏览器/服务端会话管理
  2. 用 Proxy 做路由级保护
  3. 用 Server Actions 处理登录注册表单
  4. todos 表加 user_id + RLS 策略实现数据隔离
  5. 用 Zustand 维护全局用户状态
