# API 测试指南

本文档介绍了几种测试 Todo API 的便捷方法。

---

## 方法 1: VS Code REST Client（最推荐）⭐

### 安装扩展
1. 打开 VS Code
2. 点击扩展图标或按 `Ctrl+Shift+X`
3. 搜索 "REST Client"
4. 安装 `humao.rest-client` 扩展

### 使用方法
1. 打开项目中的 `api-test.http` 文件
2. 你会看到每个请求上方有 "Send Request" 链接
3. 点击即可发送请求，响应会在右侧显示

### 优点
- ✅ 直接在 VS Code 中测试
- ✅ 请求和响应历史记录
- ✅ 支持变量和环境配置
- ✅ 语法高亮和自动补全

## 方法 4: Postman / Apifox / Insomnia

### Postman 配置
1. 创建新的 Collection
2. 基础 URL: `http://localhost:3000/api`
3. 添加请求:

| 请求名 | 方法 | URL |
|--------|------|-----|
| 获取待办 | GET | /todos |
| 创建待办 | POST | /todos |
| 更新待办 | PATCH | /todos/{id} |
| 删除待办 | DELETE | /todos/{id} |
| 获取子任务 | GET | /todos/{id}/subtasks |
| 创建子任务 | POST | /todos/{id}/subtasks |
| 更新子任务 | PATCH | /subtasks/{id} |
| 删除子任务 | DELETE | /subtasks/{id} |
| 获取附件 | GET | /todos/{id}/attachments |
| 上传附件 | POST | /todos/{id}/attachments |
| 删除附件 | DELETE | /attachments/{id} |

### 优点
- ✅ 强大的测试功能
- ✅ 环境变量管理
- ✅ 请求集合管理
- ✅ Mock 服务器支持

## 方法 5: Curl 命令

### 基础请求示例

```bash
# 获取待办列表
curl http://localhost:3000/api/todos

# 获取所有标签
curl http://localhost:3000/api/todos?action=tags

# 创建待办
curl -X POST http://localhost:3000/api/todos \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Curl 测试任务",
    "priority": "high",
    "tags": ["curl", "test"]
  }'

# 更新待办
curl -X PATCH http://localhost:3000/api/todos/{id} \
  -H "Content-Type: application/json" \
  -d '{"completed": true}'

# 删除待办
curl -X DELETE http://localhost:3000/api/todos/{id}

# 获取子任务
curl http://localhost:3000/api/todos/{id}/subtasks

# 创建子任务
curl -X POST http://localhost:3000/api/todos/{id}/subtasks \
  -H "Content-Type: application/json" \
  -d '{"name": "子任务"}'

# 更新子任务
curl -X PATCH http://localhost:3000/api/subtasks/{id} \
  -H "Content-Type: application/json" \
  -d '{"completed": true}'

# 删除子任务
curl -X DELETE http://localhost:3000/api/subtasks/{id}

# 获取附件
curl http://localhost:3000/api/todos/{id}/attachments

# 删除附件
curl -X DELETE http://localhost:3000/api/attachments/{id}
```

### 使用 jq 美化输出
```bash
# 安装 jq (JSON 格式化工具)
# Windows: scoop install jq
# macOS: brew install jq
# Ubuntu: apt install jq

curl http://localhost:3000/api/todos | jq '.'
```

## 方法 6: JavaScript 测试代码

### 在浏览器控制台测试
打开浏览器开发者工具 (F12)，在 Console 中输入：

```javascript
// 获取待办列表
fetch('/api/todos?page=1&pageSize=10')
  .then(r => r.json())
  .then(console.log);

// 创建待办
fetch('/api/todos', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    name: '控制台测试',
    priority: 'medium',
    tags: ['test', 'console']
  })
}).then(r => r.json()).then(console.log);
```

## 常见问题

### Q: 提示 "Unauthorized"
A: 需要先登录系统，在 `/login` 页面登录后才能访问 API。

### Q: 上传附件失败
A: 检查 Supabase Storage 中的 `todo-attachments` bucket 是否已创建并配置了正确的 RLS 策略。

### Q: 子任务或附件功能不工作
A: 确保已运行数据库迁移，检查 `sub_tasks` 和 `todo_attachments` 表是否存在。

### Q: 响应显示 "Todo not found"
A: 检查 ID 是否正确，确保该待办属于当前登录用户。
